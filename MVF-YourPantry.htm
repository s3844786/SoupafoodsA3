Your Pantry

 

Introduction:

In Your Pantry, users can search for nearby supermarkets, and then browse the food and products being sold, and can see the detailed information of each food, such as name, grade, unit price, and production date. Users can use the search bar to quickly find the food they want to buy, click the "Refresh" button, return to the main interface, click "more information" to view the best time to eat the food, or they can complete the purchase directly. We provide users with the information they want to know and the operations they want to perform.

 

In addition, Your Pantry has another small function, History. This function is responsible for recording the list of foods that users have purchased online or offline, helping users better manage their consumption records (users are required to upload information manually).

 

 

Your Pantry (demo):

 

1.When the user opens Your Pantry on the main interface of the application, they will see the picture below. At the top of the table, the instructions for using this function are marked.

2.The user needs to click the "load" button. At this time, the system will use the GPS function to search for nearby supermarkets. Then, the user needs to manually select a supermarket within the search range.

   

 

3.The table shows that the user selects the food and products that the supermarket is selling, and has a detailed order. Users can clearly see the name, grade, unit price (weight/prices), and production date of the food.

 

4.When the user selects a specific row of food information, click the button "More Information" on the left to view the best use time of the food, which is convenient for the user to purchase according to their needs and prevent waste.

  

 

5.Users can also enter keywords in the search bar above the form and click the "Search" button to complete a quick search.

   

6.When the user wants to return to the main interface with all product information from the "quick retrieval function", just click the "Refresh" button to complete the interface refresh.

    

 

7.When the user decides to buy, select the line of information that he wants to buy food, and then click the button "Buy". At this time, the system will ask the user what weight he needs to buy. For example: the user clicks "Apple" and then clicks the button "Buy", the system interface displays "Please enter the weight to be purchased:", the user enters 2 (this means buying 2KG), and click confirm. Then, the system will prompt the user that the purchase has been completed and display the price of the purchased food.

    

    

 

 

Your Pantry---History (demo):

1.When users use the History function, they will see the picture below. This is a function to manually upload purchase records. Users need to enter the specific purchase date, name, quantity (Weight) and price.

 

2.When the user enters "06/10/2020,apple,12,1.99", it means that he or she bought 12 apples on October 6, 2020, which cost 1.99dollars. Click "Determine" to complete the upload.

    

 

3.When users want to view the shopping records of a specific day, they only need to enter the correct time in the search bar and click the button "Search" on the left to search the shopping records of the specified day in the system.

    

 

4.When the user wants to return from the history record of the specified date, click the button "Refresh", and you can see all the shopping records uploaded before.

   

 

5.When the user finds an error in a certain line of information, he wants to delete it. Just select the line of information you want to delete and click the button "Remove" to complete the deletion.

   

 

6.The user finds an error in a certain line of the purchase record and wants to modify it, select the line of information to be modified, click the "Edit" button, enter the correct information in turn, and complete the modification.
